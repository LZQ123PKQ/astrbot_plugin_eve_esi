三、战术/战略舰船加成

战术驱逐舰（TacticalDestroyer）归类到对应阵营的战术驱逐舰操作：
- TacticalDestroyerAmarr     → 艾玛战术驱逐舰操作
- TacticalDestroyerCaldari   → 加达里战术驱逐舰操作
- TacticalDestroyerGallente  → 盖伦特战术驱逐舰操作
- TacticalDestroyerMinmatar  → 米玛塔尔战术驱逐舰操作

战略巡洋舰（StrategicCruiser）归类到对应阵营的战略巡洋舰操作：
- StrategicCruiserAmarr     → 艾玛战略巡洋舰操作
- StrategicCruiserCaldari   → 加达里战略巡洋舰操作
- StrategicCruiserGallente  → 盖伦特战略巡洋舰操作
- StrategicCruiserMinmatar  → 米玛塔尔战略巡洋舰操作

四、角色加成（roleBonus）
--------------------------------------------------
roleBonus / roleBonusOverheatDST / roleBonusCBC 等 → 特有加成

五、不显示的加成（应跳过）
--------------------------------------------------
entosisCPUPenalty (entosisCPUAdd相关) → 不显示，entosis的CPU惩罚
covertOpsWarpResistance (warpFactor相关) → 不显示，跃迁抗性被动能力

六、能量炮台大小规律
--------------------------------------------------
| 舰船类型 | 后缀 | 能量炮台大小 |
|---------|------|------------|
| 护卫舰/驱逐舰 | F/D | 小型能量炮台 |
| 巡洋舰 | C | 中型能量炮台 |
| 战列舰 | B | 大型能量炮台 |

战列巡洋舰(ABC/CBC/GBC/MBC)比较特殊，可能同时有中型和大型：
- 看modifying_attribute是否包含`bcLarge` → 大型能量炮台
- 看Effect名字是否包含`Large` → 大型能量炮台
- 其他情况默认中型

部分Effect名称中也会标注大小：
- Small / SET / SmallEnergy → 小型
- Medium / METC → 中型
- Large / bcLarge → 大型

七、Operator操作符含义（ESI返回）
--------------------------------------------------
| operator值 | 含义 | 说明 |
|-----------|------|------|
| 0 | PreAssignment | 预赋值，消耗/需求类属性用 (1-value)×100 计算降低百分比 |
| 1 | Assignment | 赋值（直接显示数值） |
| 2 | Add | 加上一个绝对值（取绝对值显示） |
| 4 | Multiply | 乘以一个倍数（取绝对值显示） |
| 6 | PercentAdd | 百分比加成（取绝对值显示） |

八、数值正负和符号规律
--------------------------------------------------
```python
def get_display_text(modified_attr_name, operator, value):
    abs_value = abs(value)

    # operator=0 - 消耗/需求类用 (1-value)×100 计算
    if operator == 0:
        if modified_attr_name in ['power', 'cpu', 'capacitor']:
            return f"{(1 - value) * 100}% 需求降低"
        return f"{value} (直接赋值)"

    # operator=1 - 直接显示数值
    if operator == 1:
        return f"{value} (直接赋值)"

    # operator=2/4/6 - 取绝对值显示
    # 1. 消耗/周期/时间类
    if modified_attr_name in ['duration', 'capacitorNeed', 'cpu', 'power']:
        return f"{abs_value}% 消耗减少"
    # 2. 病毒强度
    if modified_attr_name == 'virusStrength':
        return f"{abs_value}＋ 病毒强度加成"
    # 3. 抗性类
    if 'Resonance' in modified_attr_name or 'Resistance' in modified_attr_name:
        return f"{abs_value}% 抗性加成"
    # 4. 其他
    return f"{abs_value}% 加成"
```

常见modified_attribute分类：
| 类别 | 属性名 | 显示 |
|------|-------|------|
| 消耗/需求 | capacitorNeed, cpu, power, capacitor | 消耗减少 |
| 时间/周期 | duration, reloadTime, chargeRechargeTime | 时间减少 |
| 强度/容量 | baseSensorStrength, capacity, capacitorCapacity | 加成 |
| 伤害/倍率 | damageMultiplier, droneDamage | 伤害加成 |
| 射程/距离 | maxRange, falloff, optimalRange | 射程加成 |
| 速度/追踪 | maxVelocity, speed, trackingSpeed | 速度加成 |
| 抗性 | *EmDamageResonance等 | 抗性加成 |
| 病毒 | virusStrength | ＋加成 |

九、判断优先级
--------------------------------------------------
1. eliteBonusGunship           → 突击护卫舰操作
2. eliteBonusInterceptor       → 截击舰操作
3. eliteBonusCovertOps         → 隐形特勤舰操作
4. eliteBonusLogiFrig          → 后勤护卫舰操作
5. eliteBonusElectronicAttack   → 电子攻击舰操作
6. NavyDestroyer*     → 各阵营海军驱逐舰操作
7. TacticalDestroyer* → 各阵营战术驱逐舰操作
8. StrategicCruiser*  → 各阵营战略巡洋舰操作
9. *BC（ABC/CBC/GBC/MBC）→ 各阵营战列巡洋舰操作
10. *C（AC/CC/GC/MC）   → 各阵营巡洋舰操作
11. *D（AD/CD/GD/MD）   → 各阵营驱逐舰操作
12. *F（AF/CF/GF/MF）   → 各阵营护卫舰操作
13. *B（AB/CB/GB/MB）   → 各阵营战列舰操作
14. 其他                    → 特有加成

不显示的加成（需跳过）：
- entosisCPUPenalty → entosisCPUAdd相关
- covertOpsWarpResistance → warpFactor相关
